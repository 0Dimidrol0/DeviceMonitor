Android telemetry for heavy workloads

Observe device health in real time.

DeviceMonitor is a lightweight Android library for tracking CPU, RAM, storage, battery, thermal status, network traffic, and frame timing. Built for apps that run exports, media processing, long background jobs, archive generation, and other CPU-intensive workloads.

Quick start Kotlin 
DeviceMonitor.init(context)

val monitor = DeviceMonitor.getInstance()
monitor.start()

lifecycleScope.launch {
    monitor.snapshots.collect { snapshot ->
        Log.d("DeviceMonitor", "CPU usage: ${snapshot.cpuUsagePercent}")
        Log.d("DeviceMonitor", "Battery temp: ${snapshot.batteryTempC}")
    }
}

Core telemetry

Track CPU, RAM, storage, battery, charging state, thermal throttling, and network type with a lightweight Android-first API.

Reactive streams

Expose live telemetry and warning events through Kotlin Flows so your UI, logs, or backend pipeline can react in real time.

Extended signals

Collect thermal zones, network traffic deltas, battery power data, and frame metrics for deeper device diagnostics.

Configurable sampling

Tune sampling cadence, thresholds, and enabled sensors with DeviceMonitorConfig to match your workload.

Installation

Add DeviceMonitor from Maven Central and initialize it once in your application process.

dependencies {
    implementation("io.github.0dimidrol0:DeviceMonitor:0.2.0")
}

Runtime configuration

Adjust thresholds and sampling cadence for production diagnostics, QA stress testing, or long-running background jobs.

DeviceMonitor.init(
    appContext,
    DeviceMonitorConfig.builder()
        .samplePeriodMs(5_000L)
        .memoryThresholdMb(512)
        .batteryTemperatureThresholdC(48f)
        .enableCpu(true)
        .build()
)

Extended telemetry

thermalZones gives you per-zone thermal readings from /sys/class/thermal/thermal_zone*.

networkTraffic captures TX/RX totals and deltas across time windows.

batteryPower exposes current, charge counter, and battery percentage.

frameMetrics helps profile frame duration and jank on Android 7+.

thermalZones and networkTraffic are easy to serialize to JSON for sending to your backend.

FrameMetrics hook

Android 7+ 
override fun onCreate(savedInstanceState: Bundle?) {
    super.onCreate(savedInstanceState)
    DeviceMonitor.init(applicationContext)
    if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.N) {
        DeviceMonitor.getInstance().registerFrameMetrics(window)
    }
}

override fun onDestroy() {
    DeviceMonitor.getInstance().unregisterFrameMetrics()
    super.onDestroy()
}
